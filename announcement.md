有從**合作模式(四人存檔)**搬到專用伺服器的島主,可能遇到這兩個狀況:

- 世界設定改了卻**沒有生效**

## 目前版本的解法

1. 到「**存檔備份**」分頁,點該世界的「**瀏覽檔案**」開啟存檔資料夾
2. 把裡面的 **`WorldOptions.sav`** 刪除(或改名備份)
3. **重啟伺服器**,世界設定與管理員密碼就會正常生效

## 下個版本

會**自動偵測並提供一鍵修正**:匯入共玩存檔時自動處理,既有世界也會在存檔頁顯示警告與修復按鈕。

title: Co-op (4-player) saves on a dedicated server: settings not applying / admin password mismatch
lang: en
---

If you migrated a **co-op (4-player) save** to a dedicated server, you may have hit these issues:

- World settings changes **don't take effect**
- You've set an admin password, but the player list shows "**admin password may not match**"

## The cause

Co-op saves leave behind a `WorldOptions.sav` file. When the dedicated server finds it, it **applies that file and ignores the GUI-managed config** (including the admin password) — so everything you change in the GUI gets overridden.

## The fix (current version)

1. Open the "**Saves & Backups**" tab and click "**Browse files**" on that world to open its save folder
2. Delete (or rename) the **`WorldOptions.sav`** inside
3. **Restart the server** — world settings and the admin password will now apply properly

## Next version

Will **detect this automatically and offer a one-click fix**: handled automatically when importing co-op saves, and existing worlds will show a warning with a repair button on the Saves tab.

Sorry for the trouble, and thanks for the reports!

title: 協力プレイ(4人)セーブを専用サーバーへ:設定が反映されない/管理者パスワード不一致の対処法
lang: ja
---

**協力プレイ(4人セーブ)**を専用サーバーへ移行した方は、次の問題に遭遇しているかもしれません:

- ワールド設定を変えても**反映されない**
- 管理者パスワードを設定したのに、プレイヤー一覧に「**管理者パスワードが一致しない可能性**」と表示される

## 原因

協力プレイのセーブには `WorldOptions.sav` が残っており、専用サーバーはこれを見つけると **GUI が管理する設定ファイルより優先して適用**します(管理者パスワードを含む)。そのため GUI での変更がすべて上書きされてしまいます。

## 現バージョンでの対処法

1. 「**セーブ/バックアップ**」タブでそのワールドの「**ファイルを見る**」からセーブフォルダを開く
2. 中の **`WorldOptions.sav`** を削除(または改名して保管)
3. **サーバーを再起動** — ワールド設定と管理者パスワードが正しく反映されます

## 次のバージョンでは

**自動検出とワンクリック修正**に対応します:協力プレイセーブのインポート時に自動処理し、既存ワールドにもセーブタブで警告と修復ボタンを表示します。

ご不便をおかけしました。報告ありがとうございます!

lang: en
---

First, update the GUI to **v2.2.6 or later** (auto-update does this for you).

Then open **Overview → Game Version** — you'll see a **new gray "Update Now" button**. It used to appear only when a version gap was detected; it's now **always there**.

## How to use it

1. Stop the server
2. Click the gray "**Update Now**" → confirm → let the progress bar finish

## Why

- **Some networks can't reach Steam**, so no version gap is ever detected and the old button never appeared — now you can force an update regardless
- The update itself **includes file validation**, so it doubles as a repair; it's safe to press even when already up to date
- Combined with the v2.2.5–2.2.6 fixes, updates now **automatically terminate** crash-leftover processes locking server files (CrashReportClient etc.) — if your updates kept failing with `0xE0434352`, this button should now succeed

If it still fails, the error message shows the cause inline (with diagnostic output) — follow the hint or bring it to our Discord.
