# palserver GUI

[繁體中文](README.md) | [简体中文](README.zh-CN.md) | [English](README.en.md) | **日本語**

<p align="center"><a href="https://palserver-GUI.iosoftware.ai"><b>公式サイト palserver-GUI.iosoftware.ai</b></a> —— ダウンロード・ガイド・FAQ</p>

**Palworld(パルワールド)専用サーバーの GUI 管理ツール。**
ホストマシンで agent を 1 つ動かし、あとはブラウザからサーバーを管理 —— サーバー起動、設定変更、プレイヤー確認、セーブのバックアップ、MOD 導入まで、コマンドラインは一切不要です。

スマホ・タブレット・別の PC からも管理でき、リンク 1 本で友だちを共同管理者として招待できます。

```
ブラウザ(React Web UI)
        │  HTTP / WebSocket(Bearer トークン)
        ▼
   agent(Node/TypeScript,Fastify)
        ├── native バックエンド(既定):ホスト上で PalServer を直接起動、Docker 不要
        └── docker バックエンド(ベータ):PalServer をコンテナ内で実行
```

## 機能一覧

**サーバー構築と管理**
- 複数のサーバーインスタンスを作成。それぞれ独立したワールド・ポート・設定を持ち、ワンクリックで起動 / 停止 / 再起動 / 削除(削除してもセーブは残ります)
- Palworld サーバーファイルの自動ダウンロード&インストール(DepotDownloader 使用)、**リアルタイムのインストール進捗バー**つき、または**既存のインストールディレクトリをそのまま引き継ぎ**
- ゲームバージョン確認:インストール済みビルドと Steam の最新版を比較し、ワンクリックでサーバー更新
- リアルタイムログストリーミング(agent / ゲーム / PalDefender の 3 ソースを切替可能)
- 起動オプションパネル:Steam クエリポート(queryport)を**自由に設定可能**(他インスタンスとの重複チェックつき);`publiclobby` / `logformat` などの起動フラグも設定に統合
- **Docker カスタムコンテナイメージ**:すでに使っている他の Palworld イメージを流用可能。docker / k8s バックエンドは**特定 OS に縛られなくなりました**(macOS/Windows でも Docker Desktop があれば利用可、k8s は常に選択可)

**ワールドとパフォーマンス設定**
- 80 以上のワールド設定を GUI で編集。カテゴリ別に整理され、型・範囲・既定値つき。生の `PalWorldSettings.ini` を直接編集することも可能
- `Engine.ini` パフォーマンス調整(Tick レート、ネットワークレート、タイムアウト、GC 間隔、パフォーマンスフラグ `useperfthreads` / `NoAsyncLoadingThread` / `UseMultithreadForDS`、ワーカースレッド数…)+ワンクリックプリセット;プレイヤー帯域幅の上限は 1 Gbps まで調整可能で、Mbps 換算をリアルタイム表示
- 設定ファイル破損の自動検出と「クリーンな設定ファイルの再生成」(壊れたファイルは先にバックアップされ、勝手に消されません)

**プレイヤー管理**
- オンラインプレイヤー一覧:レベル、Ping、座標、建築数。プレイヤーをクリックすると**手持ちのパルとインベントリ**を確認できます(PalDefender が必要)
- キック、BAN、ホワイトリスト —— **オフラインのプレイヤーにも操作可能**(BAN 解除など)
- プレイヤー履歴:agent が 15 秒ごとに記録。プレイ時間、接続回数、初回/最終ログイン、入退出タイムライン
- 全体アナウンス、即時セーブ

**マップ**
- **内蔵の完全なワールドマップ**(パルパゴス諸島 + サクラジマ + フェイブレイク、高解像度)——マップ画像を自分で用意する必要はありません
- **オンラインプレイヤーのリアルタイム表示** + **オフラインプレイヤーの最終位置**;ギルド拠点、**野生ボス(Alpha パル)レイヤー**、ランドマーク(高速移動ポイント / 塔 / ダンジョン、UI 言語に合わせた名称)
- **フルスクリーンマップ**(`/map`)、ライブマップからワンクリックで新しいタブに開ける
- **マップをクリックして座標を選択**:テレポートや生成など座標が必要なコマンドは、数値入力の代わりにピンを置くだけ

**コンソール**
- 検索・カテゴリ・パラメータフォームつきのフル RCON コンソール。危険なコマンドは確認ダイアログつき
- プレイヤー ID が必要なパラメータにはプレイヤーピッカー(オフライン含む)、アイテム / パル / 卵の ID にはアイコン検索
- PalDefender 導入済みなら、そのコマンドも自動で追加
- パル・アイテムデータを **Palworld 1.0(フェイブレイク)**に対応;アクティブスキル / 特性を多言語化(繁体字中国語 / 日本語)

**セーブデータ移行(内蔵、コマンド不要)**
- **セーブデータをインポート**:「サーバーを作成」の隣のボタンから、既存のワールドを引き継いで新規サーバーを作成 —— 3 種類のソースに対応:**他の専用サーバー**、**ローカルマルチのセーブ**(4 人招待コード)、**旧 v1.0 GUI**。フォルダパスを貼り付け → スキャン → ワールドを選択するだけで、インポート前に自動バックアップし、そのワールドを自動でアクティブに設定
- **ホストキャラクター修復**(内蔵の `palworld-host-save-fix`):ローカルマルチのセーブを専用サーバーに移すと、ホストはキャラクターの再作成を求められますが、「セーブ・バックアップ」タブがマルチのホストセーブを検出してワンクリックで引き継ぎ(Python 不要)。新形式の **PlM(Oodle)セーブフォーマット**にも対応し、修復前に必ず自動バックアップ
- インポート後に新しく追加されたキャラクターファイルは自動で**「インポート後に追加」**とタグ付けされ、あらかじめ選択された状態になるので、どれがホストの新キャラクターか迷いません
- 移行の詳しい手順:[docs/MIGRATION.md](docs/MIGRATION.md)

**セーブとバックアップ**
- スケジュール自動バックアップ:間隔、保持数、無人時のスキップ
- 手動バックアップ / 復元 / ダウンロード。復元前には現在のワールドを自動でバックアップ
- マルチワールド管理:全ワールドの一覧、アクティブワールドの切替、個別プレイヤーセーブの削除;プレイヤーキャラクターファイルの一覧はリアルタイムで更新

**MOD**
- **PalDefender**(アンチチート、旧 Palguard)と **UE4SS**(Lua/Blueprint MOD ローダー)をワンクリックでインストール / 更新 / 削除。安定版とベータ版のチャンネルあり
- PalDefender 設定パネル、Lua MOD の ON/OFF、pak MOD 管理;**PalDefender の REST API ポートを変更可能**
- **MOTD ログイン告知**が設定画面から編集可能に
- ファイルマネージャ:サーバーディレクトリのファイルを閲覧・アップロード・編集・削除

**安定運用**
- 自動再起動:スケジュール(一定間隔または毎日指定時刻)、メモリしきい値、クラッシュ復旧(1 時間あたりの上限つきで無限再起動ループを防止)
- 計画再起動の前にはカウントダウンを配信してセーブ。手動停止はクラッシュ扱いになりません

**スポンサー限定機能**(有効なスポンサーが解放)
- **パルステータスエディタ**(PalSchema 経由):種族の基礎ステータス HP / 攻撃 / 防御 / 捕獲率などを変更、**ボス個体は個別に調整可能**;PalSchema のワンクリックインストール、変更履歴一覧、ワンクリック全復元
- **プレイヤーのテレポート**:プレイヤーを別のプレイヤーの位置へ、または**マップでクリックした座標**へ転送
- **アイテムの一括付与**:アイテムアイコン選択 + 個数指定で、複数種類をまとめて付与
- **配合計算**:セーブスキャンで取得したサーバー全体のパルから、目標種族とパッシブスキルへの最短配合ルートを計算。ツリー表示で各ステップの両親個体・所有者・所在を確認できます(レシピデータは MIT ライセンスの Pal Calc 由来)
- カスタムパル / パルの卵、ギルド拠点の詳細、ランドマーク名の編集

**その他**
- 4 言語対応:繁體中文 / 简体中文 / English / 日本語。**6 種類のテーマ**(パルワールドオリジナル / シルバー / オーロラエメラルド / ミッドナイトライラック / サクラピンク / オレンジキャット)**×** ライト / ダーク —— 一部のテーマはスポンサー限定
- ホーム画面のサーバーカードを**ドラッグで並べ替え**;タブは**表示 / 非表示をカスタマイズ**可能;概要カードは非表示にできます
- 接続診断:公開 IP や NAT/CGNAT 配下かを検出し、VPN(Tailscale / Radmin)での公開手順を案内
- GUI 自己アップデート(任意):GitHub Releases を確認し、SHA256 検証後にバイナリを差し替えて自動再起動

## 動作要件

| 項目 | 説明 |
| --- | --- |
| **OS** | **Windows 10+ または Linux(x86_64)**。macOS は agent は動きますが **Palworld サーバーは動きません**(SteamCMD/PalServer 非対応)。開発やリモートホスト管理用のみ。 |
| **ハードウェア** | Palworld 公式要件に準拠。サーバーファイルは数十 GB あり、初回インストールには時間がかかります |
| **Node.js** | **不要**(ポータブル実行ファイルに同梱)。ソースから動かす場合のみ Node 20+ と pnpm が必要 |
| **Docker** | 不要。docker バックエンド(ベータ)を使う場合のみ |

## プレイヤー向け:5 分でサーバーを立てる

> 図解つき完全ガイド(友だちの招待、VPN 設定):**[公式サイト](https://palserver-GUI.iosoftware.ai)** と **[FAQ](https://faq.toc.icu/)**

1. [Releases](https://github.com/io-software-ai/palserver-gui/releases) から OS に合ったアーカイブ
   (`palserver-agent-windows.zip` / `-linux.zip`)をダウンロードして展開。
2. 中の `palserver-agent`(Windows は `palserver-agent.exe`)を実行。Node も Docker も不要です。
3. ウィンドウの案内に従って **`http://localhost:8250`** を開きます。ローカル管理に**パスワードは不要**。
4. 「サーバーを作成」をクリック。初回は Palworld サーバーファイルをダウンロードします(**数十 GB、気長に**)。画面にリアルタイムの進捗バーが表示されます。
5. インストールが済んだら「起動」を押せば開設完了。

**すでに既存のワールドがある?** 「サーバーを作成」の隣にある「セーブデータをインポート」から、他のサーバーやローカルマルチ(4 人招待コード)、v1 の GUI のワールドを新しいサーバーに引き継げます。詳しくは[セーブデータ移行ガイド](docs/MIGRATION.md)を参照してください。

**友だちを共同管理者に:** 起動ウィンドウに `?setup=XXXX-XXXX` 形式のリンクが表示されます。相手のブラウザで開いてもらえば接続できます
(同じ LAN か VPN 内にいる必要があります)。agent の URL を開いて**ペアリングコード**を入力してもらう方法もあります。

**友だちをゲームに招く:** いちばん簡単なのは VPN(Tailscale か Radmin)。GUI の接続カードがネットワーク環境を検出して手順を案内します。
公開 IP があれば従来どおりのポート転送(UDP 8211)でも OK。

> **マップについて:** GUI には完全なワールドマップ(パルパゴス諸島 / サクラジマ / フェイブレイク)が内蔵されているので、
> マップ画像を自分で用意する必要はありません。「マップ」タブか、フルスクリーンの `/map` を開けば、オンラインプレイヤーの
> リアルタイム位置、オフラインプレイヤーの最終位置、ギルド拠点、野生ボスが確認できます。

## 管理者向け:運用ガイド

### セキュリティモデル

agent の入口はひとつだけ:**ループバックは認証不要、それ以外はすべてトークン必須。**

- **ローカル管理**(`127.0.0.1`)は認証情報不要 —— 1 台で使う分には摩擦ゼロ。
- **他のデバイス**は API トークン(`Authorization: Bearer <token>`)を付けるか、**ペアリングコード**でトークンを取得します。
  ペアリングコードは読みやすい `XXXX-XXXX` 形式(紛らわしい文字は除外)。いつでも再生成でき、旧コードとリンクは即座に無効化されます。
- トークンはデータフォルダに保存(権限 `0600`)。初回起動時に生成され、ウィンドウに表示されます。
- 共用マシンでは `PALSERVER_REQUIRE_TOKEN=1` を設定すると、ループバックでもトークン必須になります。
- **SteamID を全面的にマスク**:名簿・ログ・プレイヤーピッカー・コマンド出力などは常に一部を伏せた表示(クリックで表示 / コピー可能)。
  ペアリングコードとワンクリックログインリンクは既定で**モザイク表示**になり、スクリーンショットからの流出を防ぎます。

> agent はホストのファイルとプロセスを直接操作します。**`:8250` をそのまま公開インターネットに晒さないでください。**
> リモート管理は VPN(Tailscale/WireGuard)か、TLS つきリバースプロキシ越しで。

### 環境変数

| 変数 | 既定値 | 用途 |
| --- | --- | --- |
| `PALSERVER_DATA_DIR` | `~/.palserver-agent` | 全状態の保存先 |
| `PALSERVER_AGENT_PORT` | `8250` | 待受ポート |
| `PALSERVER_AGENT_HOST` | `0.0.0.0` | バインドアドレス |
| `PALSERVER_REQUIRE_TOKEN` | 未設定 | `=1` でループバックにもトークンを要求 |
| `PALSERVER_TLS` | 未設定 | `=1` で HTTPS 待受(自己署名証明書を `<data-dir>/tls` に自動生成。自前の証明書も可) |
| `PALSERVER_WEB_ORIGINS` | 空 | クロスオリジン接続を許可するオリジン(カンマ区切り)。独立デプロイの公開 web サイト用 |
| `PALSERVER_AUTO_UPDATE` | 未設定 | `=0` で GUI 自己アップデートを完全無効化(チェックもしない) |
| `PALSERVER_TELEMETRY` | 未設定 | `=0` で匿名利用統計を強制無効化 |
| `PALSERVER_STATS_URL` | 公式エンドポイント | 自前の統計バックエンドに変更 |
| `PALSERVER_GITHUB_REPO` | `io-software-ai/palserver-gui` | 自己アップデートが参照するリポジトリ |
| `PALSERVER_IMAGE_VANILLA` | `palserver/vanilla:latest` | docker バックエンドが使うイメージ |

### データの場所

```
~/.palserver-agent/
├── token                 API トークン(0600)
├── pair-code             ペアリングコード(0600)
├── instances.json        全インスタンスの設定(唯一の情報源)
├── tools/                キャッシュされた DepotDownloader
├── tls/                  自己署名証明書(PALSERVER_TLS=1 のとき)
└── instances/<id>/
    ├── server/           agent がインストールしたサーバーファイル(既存引き継ぎ時はなし)
    ├── server.pid        ゲームプロセスの pid
    ├── server.log        agent が取得したサーバー出力
    └── backups/          tar.gz バックアップ
```

サーバープロセスは **detached** で起動されるため、agent の再起動(や自己アップデート)でゲームサーバーが**落ちることはありません**。
pid ファイルで agent が再接続します。

### デプロイ方法

**ポータブル実行ファイル(推奨)** —— プレイヤー向けと同じ方法。ほとんどの人はこれで OK。

**agent 自体を Docker で動かす**(Linux ホスト):

```sh
docker compose up -d          # docker-compose.yml 参照
```

`docker.sock` のマウントが必要で、ホスト側のフォルダパスをコンテナ内と一致させる必要があります(インスタンスディレクトリはゲームコンテナに bind-mount されます)。

**web サイト単体 + リモート agent** —— Release の `palserver-web.zip` は独立デプロイ可能なフロントエンドです。サイトのオリジンを agent の
`PALSERVER_WEB_ORIGINS` に追加すれば、公開サイトから自宅の agent に接続できます。

**ソースから** —— 下記の[開発ガイド](#開発者向け開発ガイド)参照。`pnpm release:exe` で自分用のポータブル実行ファイルを作れます。

### 自己アップデート

「設定 → GUI アップデート」にあります。既定では**チェックのみでインストールはしません**(6 時間ごと)。新版が見つかると更新カードが表示され、クリックするまで何もしません:
プラットフォーム別の `.tar.gz` をダウンロード → **`SHA256SUMS.txt` と照合** → 実行ファイルとフロントエンドを差し替え → 自動再起動。「自動インストール」も有効化できます。

安全設計:チェックサムファイルがなければ更新を拒否。ポータブル実行ファイル以外(開発モードなど)では自己アップデートを拒否。サーバーのインストール進行中も拒否
(ダウンローダーは agent の子プロセスのため、再起動すると中断されます)。差し替え失敗時は旧実行ファイルを復元します。

### プライバシーと匿名統計

GUI は**匿名**の利用カウント(インストール数、サーバー作成/起動数、ユニークプレイヤー数)を送信し、利用規模の把握に使います。
個人情報・IP・サーバー名・セーブ内容は含まれません。プレイヤー識別子は一方向ハッシュのみ送信。
設定で無効化できるほか、`PALSERVER_TELEMETRY=0` で強制無効化も可能。詳細:**[PRIVACY.md](PRIVACY.md)**。

## 開発者向け:開発ガイド

### アーキテクチャ

フロントエンドはゲームの REST API・RCON・PalDefender の API に**直接触れません** —— それらの認証情報は agent の中だけに留まり、ブラウザは agent とだけ通信します。

| パッケージ | 内容 |
| --- | --- |
| `packages/agent` | Fastify デーモン:REST + WebSocket API、プロセス管理、RCON、バックアップ、MOD インストール、自己アップデート |
| `packages/web` | React 18 + Vite + Tailwind 4 の Web UI |
| `packages/shared` | 共有の zod スキーマと API 型(ワールド設定、インスタンス契約) |
| `packages/stats` | Cloudflare Worker + D1、匿名統計コレクタ |
| `images/vanilla` | docker バックエンド用の Linux PalServer イメージ(DepotDownloader 同梱) |
| `images/dev-stub` | Apple Silicon 開発用の偽 PalServer |
| `deperated/` | v1 の Electron 版。UX/i18n の参考としてのみ保持、workspace 外 |

### 開発を始める

Node 20+ と pnpm 11 が必要です。

```sh
pnpm install
pnpm build

pnpm dev:agent    # ターミナル 1 — agent(初回は API トークンを表示)
pnpm dev:web      # ターミナル 2 — Web UI on http://localhost:5173
```

agent は既定で `:8250` を待ち受けます。`packages/web/dist` があれば agent がフロントエンドも配信します(同梱ビルド)。

| コマンド | 内容 |
| --- | --- |
| `pnpm typecheck` | workspace 全体の型チェック(CI で実行) |
| `pnpm build` | 全ビルド |
| `pnpm bundle:agent` | esbuild で単一 CJS にバンドル |
| `pnpm release:exe` | 現在のプラットフォーム向けポータブル実行ファイルを `release/` に生成 |

### ワールド設定はスキーマ駆動

`packages/shared/src/options.ts` が**唯一の情報源**です:各オプションの型・既定値・範囲・カテゴリがここにあります
([公式ドキュメント](https://docs.palworldgame.com/)と照合済み)。zod スキーマ、agent の ini シリアライズ、フロントエンドの設定エディタはすべてここから導出されます ——
**ここにオプションを 1 つ足せば、パイプライン全体が通ります**。中国語ラベルは `packages/web/src/labels.ts`。

`Engine.ini` と PalDefender の `Config.json` も同じ方式で、**書き込みはマージ**です:GUI が管理しないセクション・キー・コメントはそのまま保持されます。

### i18n

コード内の文字列は**中国語の原文**で書き、`t("中文")` が原文をキーに辞書を引きます。
`packages/web/public/i18n/{en,ja,zh-CN}.json` が「中国語 → 訳文」の対照表で、見つからなければ中国語原文を表示するため、**未翻訳でもレイアウトは壊れません**。
辞書はバックグラウンドで GitHub raw から最新版を取得するので、翻訳修正にリリースは不要です。

### Apple Silicon での開発

実サーバーは Rosetta では動きません(SteamCMD は 32-bit、PalServer はワールドセーブ時に segfault)。UI/agent の開発には偽サーバーを:

```sh
docker build -t palserver/dev-stub:latest images/dev-stub
PALSERVER_IMAGE_VANILLA=palserver/dev-stub:latest pnpm dev:agent
```

実サーバーでの検証には x86_64 の Windows か Linux マシンが必要です。

### リリース

`v*` タグを push すると [release workflow](.github/workflows/release.yml) が 3 つの OS でそれぞれ生成します:

- `palserver-agent-<os>.zip` —— 手動ダウンロード用
- `palserver-agent-<os>.tar.gz` —— 自己アップデート用
- `palserver-web.zip` —— 独立デプロイ可能なフロントエンド
- `SHA256SUMS.txt` —— 自己アップデートが必ず検証
